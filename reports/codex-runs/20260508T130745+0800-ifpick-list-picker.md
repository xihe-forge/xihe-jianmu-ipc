# ifpick list picker final report

Time: 2026-05-08T13:07:45+0800
Branch: master

## Summary

Implemented `ifpick` in `bin/install.ps1` as an interactive PowerShell session picker for recent Claude and Codex sessions. It merges Hub history, Claude project JSONLs, Codex session JSONLs, and Codex archived JSONLs so users can resume from a numbered table instead of trying `ipc` or `ipcx` names manually.

The implementation also handles the orphan Codex gap: if a Codex JSONL has no `IPC_NAME` marker because the session was started with vanilla `codex`, `ifpick` dispatches direct `codex resume <sessionId>` with the requested xhigh reasoning and bypass flags.

## Behavior Table

| Area | Exact behavior |
| --- | --- |
| Hub source | `Invoke-WebRequest -UseBasicParsing -TimeoutSec 2 http://127.0.0.1:3179/sessions-history?limit=50`; keeps non-empty `sessionId` and runtime `claude`, `codex`, or `unknown`; Hub errors fall through silently. |
| Claude filesystem source | Reads `$env:USERPROFILE\.claude\projects\D--workspace-ai-research-xiheAi\*.jsonl`, sorted by `LastWriteTime`; extracts IPC name from `[session-state-writer] throttle skip for ipc_name=<name>`. |
| Codex filesystem source | Reads `$CODEX_HOME\sessions\**\*.jsonl` or `$env:USERPROFILE\.codex\sessions\**\*.jsonl`, plus `archived_sessions\*.jsonl`; parses first-line `session_meta.payload.id` and `cwd`; extracts `IPC_NAME="<name>"` when present. |
| Merge | Union by `sessionId`; filesystem rows fill gaps; Hub rows override duplicates as authoritative. |
| Sort | Descending by Hub `lastSeenAt`, then `spawnAt`, then filesystem `LastWriteTime`. |
| Display | Top 30 by default; `IFPICK_LIMIT` overrides; columns are `#`, `runtime`, `ipc-name`, `last-seen`, `cwd-tail`, `sessionId-prefix`. |
| Input | Empty or `q` aborts; invalid/out-of-range input prints an error and re-prompts. |
| Claude dispatch | `ipc <name> -resume <sessionId>`. |
| Named Codex dispatch | `ipcx <name> -resume <sessionId>`. |
| Orphan Codex dispatch | `codex resume <sessionId> --dangerously-bypass-approvals-and-sandbox -c "model_reasoning_effort=\"xhigh\""`. |
| Dry run | `IFPICK_DRYRUN=1` prints the dispatch command and exits without spawning. |
| Dot-source support | `install.ps1` evaluates the generated function blocks after profile installation, so `. .\bin\install.ps1; ifpick` works in the current shell. |
