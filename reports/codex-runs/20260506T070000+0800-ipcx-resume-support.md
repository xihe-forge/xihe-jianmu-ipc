# ipcx -resume support

## Summary

Implemented `ipcx <name> -resume` for Codex runtime with the same operator shape as `ipc <name> -resume`:

- `ipcx <name> -resume` / `-resume 0`: resolve latest Codex session for the IPC name.
- `ipcx <name> -resume N`: resolve `HEAD~N` by IPC name.
- `ipcx <name> -resume <UUID>`: resume the explicit Codex session UUID.

Primary lookup is Hub `GET /sessions-history?name=<name>&limit=10` filtered to `runtime=codex`. Local fallback is `$CODEX_HOME\ipcx-session-map\<name>.jsonl`, then raw Codex JSONL discovery under `$CODEX_HOME\sessions`.

## Codex resume interface

Verified with `codex resume --help` on Windows:

- Usage: `codex resume [OPTIONS] [SESSION_ID] [PROMPT]`
- `--last`: continue the most recent recorded session without picker.
- Direct UUID/session id is accepted as `[SESSION_ID]`.
- Options accepted include `-c`, `--dangerously-bypass-approvals-and-sandbox`, `--cd`, `--all`, and `--include-non-interactive`.
- `codex resume <uuid> -c ... --version` parses options after the UUID and prints `codex-cli-resume 0.128.0`, so the implemented argv order is valid.

Local Codex TUI history on this Windows host is:

```text
C:\Users\jolen\.codex\sessions\YYYY\MM\DD\rollout-<timestamp>-<uuid>.jsonl
```

The first JSONL line is `type=session_meta`; `payload.id` is the CLI resume id. Codex App Server `threadId` is a separate live app-server concept used by `thread/*` JSON-RPC and is not the CLI resume id.

## Implementation

- `bin/install.ps1`
  - `ipcx` now accepts `[switch]$resume` and `[ValueFromRemainingArguments=$true)][object[]]$rest`.
  - Numeric indexes resolve via Hub first, then local Codex session map / JSONL fallback.
  - UUID resume goes directly to `codex resume <uuid>` and does not call Hub.
  - Existing Codex flags remain: `IPC_NAME`, `IPC_RUNTIME=codex`, `model_reasoning_effort="xhigh"`, and bypass approvals/sandbox.
- `bin/codex-title-wrapper.mjs`
  - Records fresh and resumed Codex session metadata to `$CODEX_HOME\ipcx-session-map\<safe-name>.jsonl`.
  - Posts Codex session history to Hub `POST /sessions/register-history` with `runtime=codex`.
  - Fixes consecutive fresh-session detection: wrapper now uses `session_meta.payload.timestamp` plus a 2s creation window so a previous fresh session is not mistaken for the next launch before the new transcript appears.
- Tests:
  - Added `tests/ipcx-resume-ps.test.ps1` for PS5/PS7 command construction, Hub priority, local fallback, UUID direct, and missing-name errors.
  - Extended `tests/codex-title-wrapper.test.mjs` to cover local map write and stale previous-session avoidance.
